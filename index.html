◆ Setup Guide

How I Connected NetSuite to AI Using MCP

A step-by-step guide for finance professionals. No developer required. No code. One afternoon.

● 6 steps ● ~30 min setup ● NetSuite 2025.2+
Before You Start
  • NetSuite admin access (or get your admin involved for steps 1-3)
  • An AI client that supports remote MCP servers (Claude Pro, ChatGPT Pro/Plus, etc.)
  • NetSuite version 2025.2 or later
01
Enable Required SuiteCloud Features
Setup › Company › Enable Features › SuiteCloud

Make sure these are all enabled:

  • ✓ Server SuiteScript
  • ✓ REST Web Services
  • ✓ OAuth 2.0
  • ✓ Token-Based Authentication (TBA)
02
Install MCP Standard Tools SuiteApp
SuiteApps tab

In the Search SuiteApps field, search for "MCP Standard Tools" (by Oracle) and install it.

This gives you 11 tools across 4 categories:

  • ✓ Record tools -- create, read, update records
  • ✓ Report tools -- list and run financial reports
  • ✓ Saved Search tools -- list and run saved searches
  • ✓ SuiteQL tools -- run custom SQL-like queries
03
Create a Dedicated Role

Don't use Administrator -- Oracle blocks it from MCP connections. Create a new role (or clone an existing one like Accountant). Name it something like "MCP Integration Role" and set transaction permissions to View level to start read-only.

Required permissions:

PermissionLevelCategory
Log in Using OAuth 2.0 Access TokensFullSetup
MCP Server ConnectionFullSetup
REST Web ServicesFullSetup
Perform SearchFullLists
⚠
Do NOT add "OAuth 2.0 Authorized Applications Management."
Despite the similar name, this permission actively BLOCKS OAuth login. These two permissions are opposites. This was the #1 gotcha in my setup.
ⓘ
Web Services Only Role must be UNCHECKED on the role settings -- it must be unchecked for OAuth2 interactive login.
04
Assign the Role to Your User
Setup › Users/Roles › Manage Users

Find your user and add the new MCP role.

ⓘ
Make sure your user has security questions set:
Home › Settings › Update Security Questions
Missing security questions can silently block the OAuth login flow.
05
Connect Your AI Client

For Claude (claude.ai or Claude Code):

  • 1 Go to Settings › MCP Servers (or use /mcp in Claude Code)
  • 2 Enable the "NetSuite" connector
  • 3 Log in to NetSuite with OAuth2
  • 4 Select your MCP Integration Role
  • 5 Authorize the connection

For other AI clients:

Use the NetSuite AI Connector Service endpoint. Follow your client's MCP server configuration docs. The connection uses OAuth 2.0 Authorization Code Grant with PKCE.

06
Test It

Ask your AI any of these:

› List all available NetSuite reports
› Pull the trial balance for last month
› Run a SuiteQL query to show all open invoices over 90 days

If it returns data -- you're connected.

Troubleshooting

"invalid_grant" error
→
Remove "OAuth 2.0 Authorized Applications Management" from the role. Add "Log in Using OAuth 2.0 Access Tokens" instead.
Security question prompt blocks login
→
Update security questions: Home › Settings › Update Security Questions
"Administrator" role doesn't work
→
Oracle restricts Admin from MCP. Create and use a dedicated role.
No data returned
→
Check that "Perform Search (Full)" is on the role under Lists permissions.
Connection times out
→
Verify REST Web Services and OAuth 2.0 are enabled in SuiteCloud features.
